# API Testing — Urban Grocers (Postman)

## Project Overview
This project contains API testing for the Urban Grocers backend system (TripleTen training environment).

The goal was to validate API endpoints, request/response correctness, status codes, authorization, and error handling.

## Tools Used
- Postman
- REST API Testing
- Newman (CLI runner)
- Git & GitHub

## Testing Scope
- CRUD endpoint validation
- Status code verification
- Response body validation
- Negative testing
- Authorization testing
- Automated Postman tests (assertions)

## Project Structure
- `postman/` → Postman collection + environment
- `docs/` → test plan and endpoint documentation
- `newman-report/` → test execution report

## Note
This project was tested in the TripleTen training environment. The server may reset periodically, but the test artifacts and documentation remain valid.
